requirements

where possible,we would like to see mostly CSS 1&2 usage. should be limited to widely-supported elements only,or strong fallbacks should be provided.the css zen garden is about functional,practical css and not the latest bleeding-edge tricks viewable by 2% of the browsing public .the only real requirements we have is that your css validates.

luckily,designing this way shows how well various browses have implement css by now,when sticking to the guidelines you should see fairly consistent results across most modern browses .due to the sheer number of user agents on the web these days - especially when you factor in moblie - pxiel-perfect layouts may not be possible across every platform .that's okay ,but do test in as many as you can .you design should work in at least ie9+ and the latest chrome,forefox ,ios ,and andrid browses(run by over 90% of the pupolation).
